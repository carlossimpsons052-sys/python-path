Ambiente Python Moderno

Antes de escrever uma linha de código, você precisa de um ambiente bem configurado. Um ambiente bagunçado causa dores de cabeça que desmotivam — vamos fazer isso certo desde o início, com as ferramentas que o mercado usa em 2025.

python 3.12+ uv venv pyenv VSCode PATH

01 · contexto

Por que o ambiente importa tanto?

Imagine que você tem dois projetos: um usa Python 3.9 com uma versão antiga de uma biblioteca, e outro usa Python 3.12 com a versão mais recente da mesma biblioteca. Se tudo estiver instalado no mesmo lugar, eles vão entrar em conflito. Esse era o pesadelo clássico do desenvolvedor Python.

📖
A evolução do problema

Python 2 e Python 3 coexistiram por mais de 10 anos. No mesmo computador, você podia ter python apontando para Python 2.7 e python3 para Python 3.x — uma fonte enorme de confusão. Em 2020 o Python 2 foi oficialmente descontinuado. Hoje só existe Python 3, mas o problema de versões múltiplas e conflito de dependências continua.

A solução são os ambientes virtuais: pastas isoladas que contêm uma versão específica do Python e suas dependências. Cada projeto tem o seu. Quando você ativa um, tudo o que instalar vai só para aquele projeto.

02 · instalação

Instalando Python do jeito certo

A versão atual recomendada é o Python 3.12 (estável, com melhorias significativas de performance e mensagens de erro muito melhores). Evite a 3.13 por enquanto se for usar bibliotecas de ML — algumas ainda não estão totalmente compatíveis.

⚠️
Não instale pelo site oficial se for Mac/Linux

No Mac, use o Homebrew. No Linux, use o gerenciador de pacotes ou pyenv. Instalar pelo site oficial no Mac/Linux frequentemente causa problemas com o PATH e conflita com o Python do sistema.

🍎 macOS

bash 
# 1. Instale o Homebrew (se não tiver)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 2. Instale o pyenv (gerenciador de versões)
brew install pyenv

# 3. Configure o shell (cole no seu ~/.zshrc)
echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.zshrc
echo 'command -v pyenv >/dev/null || export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.zshrc
echo 'eval "$(pyenv init -)"' >> ~/.zshrc
source ~/.zshrc

# 4. Instale o Python 3.12
pyenv install 3.12
pyenv global 3.12

# 5. Verifique
python --version  # → Python 3.12.x

🪟 Windows

powershell 
# No Windows, baixe o instalador oficial
# https://www.python.org/downloads/
# ⚠️ IMPORTANTE: marque "Add Python to PATH" durante a instalação!

# Verifique depois de instalar
python --version

# Se não funcionar, tente:
py --version

🐧 Linux (Ubuntu/Debian)

bash 
sudo apt update && sudo apt upgrade -y
sudo apt install python3.12 python3.12-venv python3-pip -y

# Verifique
python3.12 --version

03 · isolamento

Ambientes virtuais com venv

O venv é o módulo nativo do Python para criar ambientes virtuais. É simples e funciona bem para projetos pequenos. Para projetos maiores ou equipes, o uv (próxima seção) é a escolha moderna.

1
Crie a pasta do projeto
mkdir meu-projeto && cd meu-projeto
2
Crie o ambiente virtual
python -m venv venv — isso cria uma pasta venv/ com Python isolado
3
Ative o ambiente
Mac/Linux: source venv/bin/activate
Windows: .\venv\Scripts\activate
Você vai ver (venv) no terminal — isso confirma que está ativo.
4
Instale dependências
pip install requests pytest — agora só existem nesse ambiente
5
Desative quando terminar
deactivate
💡
Boas práticas

Sempre adicione venv/ no .gitignore. Nunca commite o ambiente virtual. Em vez disso, salve as dependências com pip freeze > requirements.txt e quem clonar o projeto instala com pip install -r requirements.txt.

04 · o novo padrão

uv — 10x mais rápido que pip

Em 2024, a Astral lançou o uv — um gerenciador de pacotes e ambientes Python escrito em Rust. Ele substitui pip, venv, pyenv e até o pip-tools em um único binário. É drasticamente mais rápido e se tornou rapidamente o padrão da indústria.

❌ Jeito antigo
python -m venv venv
source venv/bin/activate
pip install requests pytest
pip freeze > requirements.txt
# Demora segundos a minutos
✅ Com uv (moderno)
uv init meu-projeto
cd meu-projeto
uv add requests pytest
# Pronto. Milissegundos.
# Gerencia tudo automaticamente
bash 
# Instale o uv
curl -LsSf https://astral.sh/uv/install.sh | sh  # Mac/Linux
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"  # Windows

# Crie um projeto novo
uv init meu-projeto
cd meu-projeto

# Adicione dependências
uv add requests pytest playwright

# Execute scripts sem ativar manualmente
uv run python meu_script.py
uv run pytest

# Instale uma versão específica do Python
uv python install 3.12
🎯
Por que isso importa no seu trabalho como QA

Você vai ter projetos de automação com Cypress, Playwright, Robot Framework — cada um com suas próprias dependências Python. Com uv, criar um ambiente isolado para cada projeto leva 2 segundos. Sem conflitos, sem "funciona na minha máquina".

05 · editor

VSCode configurado para Python

O VSCode é gratuito, leve e tem o melhor suporte a Python disponível. Com as extensões certas, você tem IntelliSense, debugging, formatação automática e integração com pytest — tudo funcionando junto.

Após instalar o VSCode em code.visualstudio.com, instale estas extensões:

1
Python (Microsoft)
A extensão principal. IntelliSense, linting, debugging, seleção de interpretador.
2
Pylance
Análise estática avançada. Detecta erros de tipo antes de rodar o código.
3
Ruff
Linter e formatador ultrarrápido (substitui Black + Flake8). O padrão atual.
4
Python Test Explorer
Roda e visualiza testes pytest direto no VSCode, com UI gráfica.

Configuração recomendada — crie .vscode/settings.json no seu projeto:

json 
{
  "python.defaultInterpreterPath": "./venv/bin/python",
  "editor.formatOnSave": true,
  "[python]": {
    "editor.defaultFormatter": "charliermarsh.ruff"
  },
  "python.testing.pytestEnabled": true,
  "python.testing.pytestArgs": ["tests"],
  "editor.rulers": [88]
}

06 · na prática

Seu primeiro script Python

Com o ambiente configurado, vamos confirmar que tudo funciona. Crie um arquivo hello.py e rode ele.

python — hello.py 
# Seu primeiro script Python
# Execute com: python hello.py  ou  uv run python hello.py

nome = "Carlos"
versao = __import__('sys').version.split()[0]

print(f"Olá, {nome}! 🐍")
print(f"Python {versao} está funcionando.")
print("Ambiente configurado com sucesso!")

Para rodar:

bash 
# Com venv ativado:
python hello.py

# Com uv:
uv run python hello.py

# Saída esperada:
# Olá, Carlos! 🐍
# Python 3.12.x está funcionando.
# Ambiente configurado com sucesso!
💡
F-strings são o jeito moderno de formatar texto

O f"Olá, {nome}!" é uma f-string — coloca o valor da variável dentro da string. Você vai usar isso o tempo todo. Estudaremos em detalhes no Módulo 02.

07 · contexto profissional

Como isso aparece no trabalho real

Em um projeto de automação real, você vai ter uma estrutura parecida com essa. Cada repositório de testes tem seu próprio ambiente virtual — nunca compartilhado.

estrutura de projeto real 
meu-projeto-automacao/
├── .venv/                  # ambiente virtual (no .gitignore)
├── .vscode/
│   └── settings.json       # configurações do editor
├── tests/
│   ├── api/
│   │   └── test_auth.py
│   └── e2e/
│       └── test_checkout.py
├── pages/                  # Page Objects
│   └── login_page.py
├── pyproject.toml          # dependências (com uv)
└── README.md
🎯
Fluxo de um QA Engineer ao entrar num projeto

git clone repo → cd repo → uv sync (instala tudo automaticamente) → uv run pytest (roda os testes). Em menos de 1 minuto, você está contribuindo. Isso só é possível porque o projeto tem um ambiente bem definido.

Desafio do Módulo

Nível: Iniciante

Coloque em prática o que aprendeu. Configure um ambiente do zero e escreva um script que mostre informações do seu sistema.

Tarefa: 1. Crie uma pasta chamada "pythonpath-estudos" 2. Configure um ambiente virtual com venv ou uv 3. Crie um arquivo "ambiente.py" que: - Imprime a versão do Python - Imprime o caminho do executável Python - Imprime o sistema operacional - Imprime a data e hora atual Dica: use os módulos sys, platform e datetime
python — ambiente.py 
import sys
import platform
from datetime import datetime

print("=" * 40)
print("🐍 Informações do Ambiente Python")
print("=" * 40)
print(f"Versão Python : {sys.version.split()[0]}")
print(f"Executável    : {sys.executable}")
print(f"Sistema       : {platform.system()} {platform.release()}")
print(f"Data/hora     : {datetime.now().strftime('%d/%m/%Y %H:%M:%S')}")
print("=" * 40)
print("✅ Ambiente configurado com sucesso!")
